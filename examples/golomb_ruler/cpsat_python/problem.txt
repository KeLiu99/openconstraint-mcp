Golomb ruler with 12 marks: find integer positions mark_0 < mark_1 < ... < mark_11, mark_0 = 0, such that all 66 pairwise differences mark_j - mark_i (i<j) are distinct, minimizing mark_11 (ruler length). Known optimal length for order 12 is 85. Prior saved incumbent was 91 (feasible, not optimal). This save re-verifies the original formulation (loose domain [0,91], full minimize objective, unseeded multi-worker portfolio) with a 100s internal solve budget. An MCP experiment comparing this formulation against four alternatives (tighter domain [0,87], redundant per-mark bound-tightening, and two pure-feasibility/decision reformulations targeting 85 and 87) found that all four alternatives failed to find any solution at all within a 25-30s budget, even with dedicated CPU -- likely because valid rulers are sparse near the optimum and dropping the objective or tightening the domain removes search guidance CP-SAT relies on to find a first feasible solution. A shorter unseeded run of this same original formulation once found length 90, but did not reproduce it on a subsequent unseeded re-run (both 91 and 90 have been observed). Status is "feasible", not "optimal" -- treat the saved objective as best found so far under this time budget, not a proof; the true optimum remains 85.

Golomb ruler with 12 marks: find integer positions mark_0 < mark_1 < ... < mark_11, mark_0 = 0, such that all 66 pairwise differences mark_j - mark_i (i<j) are distinct, minimizing mark_11 (ruler length). The known optimal length for order 12 is 85. This save re-verifies the original formulation (loose domain [0,91], full minimize objective, unseeded multi-worker portfolio) with a 100s internal solve budget. An MCP experiment earlier compared this formulation against four alternatives (tighter domain [0,87], redundant per-mark bound-tightening, and two pure-feasibility/decision reformulations targeting 85 and 87); all four alternatives failed to find any solution at all within a 25-30s budget, even with dedicated CPU -- likely because valid rulers are sparse near the optimum and dropping the objective or tightening the domain removes search guidance CP-SAT relies on to find a first feasible solution. This formulation's incumbent is NOT reproducible run to run under an unseeded multi-worker portfolio search: separate saves of this exact script have landed on ruler lengths of 85 (the true optimum), 90, and 91, and one run found no solution at all within the budget. `status` stays "feasible" whenever a ruler is found -- CP-SAT does not close the gap to prove optimality within the 100s budget. Treat the saved `objective` as the best found under this run's time budget, not a proof, and expect a possibly different value (or an `unknown` non-save) on a fresh unseeded re-run.
